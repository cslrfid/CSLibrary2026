## VS2022 Compile Error: "Windows" namespace not found — 2026-04-02 08:47 UTC

### Problem
User reports compile errors in Visual Studio 2022:
`The type or namespace name 'Windows' could not be found`

### Root Cause
`Source/HAL/WinFormsBLE/DeviceFinder.cs` and `CodeFileBLE.cs` use WinRT APIs:
```csharp
using Windows.Devices.Bluetooth;
using Windows.Devices.Enumeration;
```
In .NET 5+, WinRT types require:
1. **Full Windows TFM** — `net10.0-windows10.0.22621.0` (not just `net10.0-windows`)
2. **Microsoft.Windows.CsWinRT** NuGet package — the C# projection that exposes WinRT as .NET types

The csproj was using `net10.0-windows` (no version) which doesn't automatically enable WinRT projections.

### Required Fix (pending — memory flush blocked write)
Update `CSLibrary2026.csproj`:
1. Change `net10.0-windows` → `net10.0-windows10.0.22621.0`
2. Add conditional CsWinRT package:
```xml
<ItemGroup Condition="'$(TargetFramework)' == 'net10.0-windows10.0.22621.0'">
  <PackageReference Include="Microsoft.Windows.CsWinRT" Version="2.0.4" />
</ItemGroup>
```
3. Update all `Condition="'$(TargetFramework)' == 'net10.0-windows'"` to `== 'net10.0-windows10.0.22621.0'`
